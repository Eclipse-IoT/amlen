Detailed Description

Documented configuration property keys.

The global variables under this namespace ending with the suffix _PROP_KEY, are the documented public property keys that are accepted by SpiderCastConfig. The corresponding variables that end with the suffix _DEFVALUE are the default values of those properties.

const bool spdr::config::BindAllInterfaces_DEFVALUE = true

Default value of BindAllInterfaces_PROP_KEY.

const String spdr::config::BindAllInterfaces_PROP_KEY = "spidercast.comm.BindAllInterfaces"

Bind all interfaces.

Optional property.

Whether to bind the server socket to all interfaces. If true, SpiderCast will accept connections from all interfaces with the specified port. If false, and a single interface is given, SpiderCast will accept connections only from that interface.

If multiple interface addresses are provided, the node will bind to all available interfaces, regardless of the value paired with BIND_ALL_INTERFACES_KEY. If a single address is provided, the node will use the value paired with BIND_ALL_INTERFACES_KEY.

Value - boolean.

Default - true; If multiple network interfaces are provided: overrides to true.

const String spdr::config::BusName_PROP_KEY = "spidercast.BusName"

Bus name.

Mandatory property.

The bus name identifies an administrative unit of nodes that connect and maintains full membership.

Bus names are hierarchical. The level separator is '/' (forward slash). Each level must contain only graphical letters (apart from '/'). Bus name must start with a '/'.

Currently only two levels are supported.

Value - a string with no white spaces, only graphical characters.

const String spdr::config::NetworkInterface_PROP_KEY = "spidercast.comm.NetworkInterface"

Node's network interfaces (IP address) and their network-ID.

Mandatory property.

One or more network interfaces (at least one). The node will declare those addresses as the addresses in which it can be contacted. The network-ID of an interface is a string identifier for the network that interface is on.

Nodes with two interfaces with the same network-ID can communicate with one another using those interfaces. An empty string implicitly means "global" scope.

Value - a string with multiple interface-scope pairs, separated by comma:
ifc1, scope1, ifc2, scope2, ... ifcN, scopeN

Each interface entry (ifcK) with either an explicit IP address (e.g. 1.2.3.4), or a host name. The (ifcK) and (scopeK) entries should not contain white spaces or commas.

Example: a single interface with a scope:
12.22.33.44, Torus-1

Example: a single interface with a global scope:
172.16.33.44,

Example: a combination of both:
172.16.33.44, , 12.22.33.44, Torus-1

The order in which the network interfaces are declared is significant. A node (i) will try to connect to another node (j) using the first IP address with a scope that matches, where each interface of i (in order) is matched against all interfaces of j (in order). For example:

  • Node1 has: IP1, ScopeA, IP2, ScopeB, IP3, ScopeC
  • Node2 has: IP4, ScopeX, IP5, ScopeA, IP6, ScopeB

Node1 will connect to Node2 on IP5, ScopeA;

Node2 will connect to Node1 on IP2, ScopeA;
