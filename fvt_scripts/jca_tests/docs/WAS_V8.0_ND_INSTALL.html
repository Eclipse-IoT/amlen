WebSphere Application Server V8.0 Network Deployment Installation Instructions

  • Install WAS V8.0 ND on p-linux
  • Install WAS V8.0 ND on linux

Install WAS V8.0 ND on p-linux (Based on mar092)

Setup

  1. cd /WAS/installables/WAS80ND
  2. mkdir DISKS
  3. mkdir IM
  4. unzip InstallationManager-PLinux-1.4.3.zip -d IM/
  5. unzip DISK*.zip -d DISKS/

Installation Manager V1.4.3

  1. Run: /WAS/installables/WAS80/IM/consoleinst.sh
  2. Follow the prompting to install Installation Manager v1.4.3
NOTE: Change the installation directory to /opt/IBM/InstallationManager.1.4.3 to avoid conflicts with other versions. It will also create /opt/IBM/IMShared and /var/ibm/InstallationManager

WebSphere Application Server V8.0

  • Edit /WAS/installables/WAS80/responsefile80.xml with the location of the repository.config file in the DISKS directory, and the directory paths to install WAS into.
  • responsefile80.xml is already set up based on installation for mar092.
NOTE: Change the installation and eclipse directories to /opt/IBM/WebSphere80/AppServer to avoid conflicts with other versions.
  1. cd /opt/IBM/InstallationManager.1.4.3/eclipse:
  2. ./IBMIM -showProgress -silent -input /WAS/installables/WAS80/responsefile80.xml -acceptLicense

Post Installation

  • Once WAS V8.0 is installed, create the WAS profile by running:
    /opt/IBM/WebSphere80/AppServer/bin/manageprofiles.sh -create -profilePath /opt/IBM/WebSphere80/AppServer/profiles/AppSrv01 -templatePath /opt/IBM/WebSphere80/AppServer/profileTemplates/default/ -profileName AppSrv01
  • Start the server:
    /opt/IBM/WebSphere80/AppServer/bin/startServer.sh server1 -profileName AppSrv01
  • Setup security:
    1. Log on to the admin console at http://10.10.10.10:9060/ibm/console
    2. Go to Security - Global Security
    3. Run the Security Configuration Wizard
    4. Select Enable Application Security, click next
    5. Select Federated Repositories, click next
    6. Enter username/password: admin/admin
    7. Click Finish
    8. Save changes
